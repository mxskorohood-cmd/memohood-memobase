# MemoBase — инструкция по использованию

Пошаговое руководство: установка, первая загрузка документа, вопросы к базе, чтение цитат, отказы, работа с коллекциями, ключи конфига, самопроверка, YouTube (видео и целые каналы), расшифровка аудио, Obsidian, гостевые коллекции, мастер настройки `/memobase setup` (Telegram) и `hermes memobase setup` (терминал), бэкапы, карта базы, диагностика проблем.

Коротко «что это» и таблицы команд/FAQ — в [`README.md`](README.md). Здесь — подробная методичка.

## 1. Установка

1. Скопируйте папку `memobase` целиком в `<HERMES_HOME>/plugins/memobase/`. На Windows `HERMES_HOME` по умолчанию — `%LOCALAPPDATA%\hermes`.
2. Запустите установочный скрипт **из этой папки** — он ставит зависимости не в системный Python, а прямо в venv hermes-agent:

   ```powershell
   .\install.ps1
   ```

   Если автоопределение venv не сработало (нестандартная установка), укажите путь явно:

   ```powershell
   .\install.ps1 -PythonPath "C:\путь\к\hermes-agent\venv\Scripts\python.exe"
   ```

   На Linux/macOS — `./install.sh` (или `./install.sh /путь/к/venv/bin/python`).

3. Проверьте, что в `~/.hermes/.env` есть три ключа:

   ```
   CLOUDFLARE_ACCOUNT_ID=...
   CLOUDFLARE_API_TOKEN=...
   COHERE_API_KEY=...
   ```

   Это готовые внешние API для эмбеддинга (Cloudflare Workers AI, модель `@cf/baai/bge-m3`) и переранжирования (Cohere `rerank-v3.5`). Без них плагин загрузится и не упадёт, но будет работать в деградированном режиме (см. раздел «Диагностика» ниже).

   Для дополнительных источников нужны свои ключи (все опциональны — без ключа соответствующий тип загрузки честно пропускается, остальное работает):

   ```
   SCRAPECREATORS_API_KEY=...   # YouTube: субтитры и списки видео каналов
   APIFY_TOKEN=...              # YouTube: запасные субтитры + скачивание аудио
   GROQ_API_KEY=...             # расшифровка аудио (Groq Whisper) — основной путь STT
   GEMINI_API_KEY=...           # запасной путь расшифровки (Gemini)
   ```

   Для расшифровки длинных аудио (больше 25 МБ) и извлечения звука из видео дополнительно нужен `ffmpeg` в PATH (на Windows подойдёт установка через WinGet — плагин находит её сам).

4. Добавьте `memobase` в `config.yaml`:

   ```yaml
   plugins:
     enabled:
       - memobase
   ```

5. Перезапустите hermes и выполните `/memobase status` — если плагин загрузился, вы увидите список коллекций (изначально пусто — «Коллекций пока нет.»).

## 2. Первая загрузка документа

```
/memobase ingest ./docs/contract.pdf pdf
```

Что произойдёт:

1. Текст извлекается из PDF (`pdfplumber`, с откатом на `pypdf` при ошибке).
2. Текст нормализуется: убирается мусор кодировки, повторяющийся колонтитул PDF, лишние пробелы.
3. Текст режется на фрагменты (по структуре документа — заголовки, абзацы, код не разрывается посередине).
4. **Каждый фрагмент проходит сканер секретов** (API-ключи, токены, приватные ключи, высокоэнтропийные строки) — если что-то найдено, фрагмент уходит в карантин и не индексируется. Об этом вам сообщат в ответе.
5. Чистые фрагменты отправляются на эмбеддинг (Cloudflare BGE-M3) и попадают в индекс (FTS5 + векторный).

Ответ выглядит так: «Загружено в коллекцию «default»: добавлено фрагментов 42, переиспользовано 0, погашено (устарели) 0.»

Загрузка URL работает так же, но `source_type` — `url`, а сам адрес сначала проверяется на SSRF (не даст загрузить `http://169.254.169.254/`, `file://`, localhost и т.п.) — как для основного запроса, так и для каждого редиректа.

Если новых фрагментов больше порога `memobase.confirm_over_chunks` (по умолчанию 500), плагин не тратит деньги на эмбеддинг молча — сначала покажет оценку стоимости и попросит повторить запрос с `confirm=true` (через `/memobase` в разговоре с моделью — модель сама передаст этот параметр; через терминал — флаг `--confirm`).

### Повторная загрузка того же источника

Если вы загрузите тот же файл ещё раз (например, после правки), MemoBase сравнивает фрагменты по хэшу содержимого:

- неизменившиеся фрагменты **переиспользуются** (не эмбеддятся заново — экономия);
- новые/изменившиеся — индексируются;
- фрагменты, которых больше нет в новой версии, **гасятся** (tombstone) — перестают участвовать в поиске, но не удаляются физически сразу.

## 3. Вопросы к базе

```
/memobase Какой срок действия договора?
```

Под капотом (`memobase_ask`):

1. Гибридный поиск находит кандидатов: полнотекстовый (FTS5, с русским стеммингом) и векторный (по смыслу).
2. Кандидаты объединяются взаимным ранжированием (RRF) и переранжируются Cohere.
3. **Гейт достаточности**: если лучший результат ниже порога уверенности — вы сразу получаете честный отказ, модель даже не вызывается.
4. Если порог пройден, вопрос разбивается на подпункты (по «и», «?», запятым), и **изолированная модель без доступа к другим инструментам** отвечает, обязательно цитируя фрагменты дословно с указанием номера `[chunk:N]`.
5. Каждая цитата в ответе модели **проверяется дословно** против настоящего текста фрагмента (нечёткое сравнение). Не подтвердившиеся цитаты выбрасываются из ответа.
6. Если после проверки не осталось ни одной подтверждённой цитаты — вы получите отказ, а не непроверенный текст.

### Как читать ответ

```
Договор действует 12 месяцев с даты подписания.

Источники:
  [chunk:17] ./docs/contract.pdf, стр./время 3, раздел «Срок действия»: «настоящий договор действует в течение 12 (двенадцати) месяцев»

Чего не хватает в базе по этому вопросу:
  - продление договора на новый срок
```

- **Источники** — список цитат: номер фрагмента, путь/URL источника, страница/раздел, сама дословная цитата. Можно открыть исходный файл на этой странице и свериться.
- **«Чего не хватает»** (gaps) — те части вопроса, на которые ответ не нашёл подтверждения в цитатах. Это не ошибка — честное указание границы того, что подтверждено.
- Если увидите пометку **«слабое совпадение с базой»** — ответ на грани порога уверенности, стоит перепроверить его особенно внимательно.
- Если увидите **«ответ в деградированном режиме»** — переранжирование Cohere было недоступно (например, ключ не работает) и плагин использовал только RRF-порядок без дополнительного переранжирования; ответ по-прежнему проходит те же проверки цитат.

### Отказы

Примеры honest-refusal текстов, которые вы можете увидеть:

- «В базе знаний «default» пока нет ничего по этому вопросу.» — поиск вообще ничего не нашёл.
- «В базе знаний «default» не нашлось уверенного ответа на этот вопрос.» — что-то нашлось, но ниже порога уверенности.
- «В базе знаний «default» не удалось найти проверяемое подтверждение ответа.» — модель процитировала, но ни одна цитата не подтвердилась дословно.
- «В базе знаний «default» не нашлось подтверждённого ответа на этот вопрос.» — модель ответила текстом вообще без цитат.

Во всех случаях это осознанное поведение, а не сбой — плагин предпочитает явный отказ непроверенному ответу.

## 4. Коллекции

Коллекция — это отдельное пространство документов и индексов (можно, например, держать «договоры» и «техдокументацию» раздельно).

```
/memobase ingest ./manual.pdf pdf docs
/memobase list
/memobase status docs
/memobase Какая версия API описана? [обычный /memobase <вопрос> использует коллекцию по умолчанию — укажите коллекцию через инструмент memobase_ask с параметром collection, если общаетесь с моделью]
/memobase delete docs
```

Имя коллекции — только латиница, цифры, `_`/`-`, до 64 символов; зарезервированные имена (`.`, `..`, `con`, `prn`, `com1`...) отклоняются.

### Изоляция для делегированных задач

Если основной агент делегирует задачу отдельному сабагенту-«библиотекарю» с меткой `[[memobase:docs]]` в начале описания задачи, этот сабагент **привязывается к коллекции `docs`** на уровне кода: любая его попытка обратиться к другой коллекции будет отклонена с явным сообщением, а не оставлена на добросовестность модели. Обычная (непривилегированная делегированию) сессия ничем не ограничена и может обращаться к любой коллекции.

## 5. Ключи конфига (`config.yaml`, секция `kb:`)

```yaml
kb:
  embedder:
    provider: cloudflare
    model: "@cf/baai/bge-m3"
    dims: 1024
  rerank:
    provider: cohere
    model: rerank-v3.5
    enabled: true
  answer_model: ""          # пусто = активная модель хоста; можно указать дешёвую отдельную модель
  confirm_over_chunks: 500  # порог подтверждения перед платным эмбеддингом
  monthly_ceiling_usd:
    cloudflare: 5
    cohere: 5
  default_collection: default
  chunk:
    target_tokens: 900
    overlap_pct: 0.15
  canonical_host: true      # выключите на не-основном хосте при работе с общими коллекциями с нескольких машин
  youtube:
    transcript_providers: [scrapecreators, apify]  # порядок попыток получить субтитры
    confirm_over_videos: 20   # канал больше этого — сначала смета и подтверждение
  stt:
    preset: groq              # основной путь расшифровки; альтернатива: gemini
  obsidian:
    auto_detect: true         # находить vault'ы по реестру Obsidian
    auto_connect: false       # никогда не подключать vault без явного действия
  enrich:
    enabled: false            # обогащение фрагментов контекстом (см. ниже)
    model: ""                 # пусто = дешёвая модель хоста
  owner_user_id: ""           # владелец базы; первый вызвавший /memobase setup занимает это место
  guest_defaults:             # квоты гостя по умолчанию (правятся через memobase_set_guest_quota)
    max_mb: 200
    max_chunks: 4000
    daily_upload_mb: 50
    daily_budget_usd: 0.50
    daily_calls: 200
  guest_rate_limit:
    calls_per_minute: 6
  backup:
    keep: 7                   # сколько снимков хранить
    disk_alert_pct: 80        # предупреждать, если диск заполнен сильнее
    off_vps_command: ""       # шаблон внешней копии, например "rclone copy {snapshot_path} remote:kb"
```

Про `memobase.enrich.enabled`: при включении дешёвая модель на этапе индексации дописывает каждому фрагменту короткую заметку «что это и из какого места документа». Заметка попадает **только** в текст для эмбеддинга — сырой текст фрагмента, полнотекстовый индекс и проверка цитат остаются нетронутыми. Промахов поиска становится меньше, загрузка — чуть дороже и медленнее. Сбой обогащения на отдельном фрагменте не прерывает загрузку: фрагмент просто индексируется без заметки.

Изменить один ключ можно точечно: `hermes config set memobase.confirm_over_chunks 200` (или через штатный механизм hermes для dotted-key записи). Эмбеддер и профиль нарезки можно переопределить и на уровне отдельной коллекции — это уже колонки таблицы `collections` в `memobase.db`, а не `config.yaml`.

## 6. Самопроверка индекса (`memobase_selfcheck`)

```
/memobase selfcheck default
```

Плагин берёт случайную выборку фрагментов, генерирует по каждому контрольный вопрос (моделью, если она доступна, иначе — эвристически, отрывком текста самого фрагмента) и проверяет, находит ли гибридный поиск исходный фрагмент по этому вопросу в топ-10. В ответе — процент покрытия, какие фрагменты не нашлись, и предложенные (но не сохранённые автоматически) пороги достаточности.

Полезно прогонять после большой загрузки — если покрытие низкое, возможно, стоит скорректировать профиль нарезки или пороги коллекции вручную.

## 7. YouTube: одно видео и целые каналы

Нужен ключ `SCRAPECREATORS_API_KEY` в `~/.hermes/.env`; для видео без субтитров — ещё `APIFY_TOKEN` и `GROQ_API_KEY`.

YouTube загружается просьбой в чате обычным текстом (слэш-команда `/memobase ingest` — только для файлов и URL):

```
Загрузи в базу знаний это видео: https://www.youtube.com/watch?v=XXXXXXXXXXX
```

Модель вызовет `memobase_ingest` с типом `youtube`. Что происходит под капотом:

1. Сначала плагин пробует забрать готовые субтитры через ScrapeCreators; если тот не ответил — через Apify (порядок задаётся ключом `memobase.youtube.transcript_providers`).
2. Если субтитров у видео нет вообще, это не ошибка: аудио скачивается через Apify (напрямую с сервера YouTube не выйдет — датацентровые адреса он блокирует) и расшифровывается Groq Whisper.
3. Каждый блок текста несёт таймкод — цитата из видео в ответе будет вести на конкретную секунду ролика.

Целый канал — так же, только ссылка на канал (`@имя`, `/channel/...`):

```
Загрузи в базу весь канал https://www.youtube.com/@MaximSkorohood
```

Что вы увидите: если видео на канале больше порога `memobase.youtube.confirm_over_videos` (по умолчанию 20), плагин **не начнёт тратить деньги молча** — сначала покажет число видео и смету в долларах (список + субтитры + аудио для роликов без субтитров + расшифровка) и попросит подтвердить. После подтверждения каждое видео становится отдельным документом.

Типовые ошибки:

- **«пропущено: нет ключа...»** — не задан `SCRAPECREATORS_API_KEY` (или `APIFY_TOKEN`/`GROQ_API_KEY` для роликов без субтитров). Добавьте ключ в `~/.hermes/.env` и повторите.
- **Загрузка канала оборвалась на середине** — просто повторите ту же просьбу: уже загруженные видео переиспользуются по хэшу содержимого, деньги повторно не тратятся.
- **Часть видео «пропущена»** — это осознанное поведение: если для конкретного ролика все пути отказали (нет субтитров и не удалось скачать аудио), он честно помечается пропущенным с причиной, а не роняет весь пакет.

## 8. Расшифровка аудио и голосовых

Нужен `GROQ_API_KEY` (основной путь) и, по желанию, `GEMINI_API_KEY` (запасной). Для файлов больше 25 МБ и для видео — `ffmpeg` в PATH.

```
Расшифруй и добавь в базу знаний файл ./записи/встреча.ogg
```

Модель вызовет `memobase_ingest` с типом `audio` (для видеофайла — `video`: сначала ffmpeg достанет звуковую дорожку). Что происходит:

1. Файл до 25 МБ уходит в Groq Whisper (`whisper-large-v3-turbo`) целиком — назад приходят сегменты с настоящими таймкодами от декодера.
2. Файл больше 25 МБ автоматически режется на ~20-минутные куски с нахлёстом, куски расшифровываются по очереди и склеиваются посегментным сопоставлением — текст не теряется и не дублируется.
3. Если Groq недоступен (лимит, ошибка), подхватывает Gemini целым файлом. Важно: таймкоды Gemini — это то, что модель «утверждает», на длинных записях они дрейфуют, поэтому такая расшифровка помечается пониженным доверием.

Что вы увидите: обычный ответ о загрузке («добавлено фрагментов N...»), а при вопросах по этой записи цитаты будут с таймкодами вместо номера страницы.

Типовые ошибки:

- **«пропущено: нет ключа GROQ_API_KEY»** — добавьте ключ (бесплатный лимит Groq для Whisper заметный) или задайте `memobase.stt.preset: gemini`.
- **Большой файл не расшифровывается** — проверьте, что `ffmpeg` доступен в PATH: без него нарезка больших файлов и видео невозможна.
- **Таймкоды «уплывают»** — посмотрите, кто расшифровывал: у Gemini-пути это известная особенность (см. выше); путь Groq даёт точные таймкоды.

## 9. Подключение Obsidian

Ключей не нужно. Плагин сам находит vault'ы по реестру Obsidian (`obsidian.json`) — это же делает мастер `/memobase setup` и сообщает, что нашёл.

```
Добавь в базу знаний мой Obsidian vault D:\Заметки
```

Модель вызовет `memobase_ingest` с типом `obsidian`. Можно указать и путь к одной заметке `.md`. Что происходит:

- читаются только `.md`-файлы; служебные папки `.obsidian/`, `.trash/`, `templates/` пропускаются;
- YAML frontmatter (теги, даты) сохраняется как метаданные, связи `[[...]]` — тоже (их видно на карте `/memobase map`);
- vault — **только для чтения**: MemoBase никогда ничего в него не пишет и никогда не подключает vault сам, без вашего явного действия (`memobase.obsidian.auto_connect: false` по умолчанию).

Что вы увидите: сводку по vault: сколько заметок всего, сколько загружено, сколько не изменилось, сколько не удалось.

Повторная загрузка того же vault (например, раз в день) дешёвая: неизменённые заметки переиспользуются по хэшу — платите только за новые и отредактированные.

Типовые ошибки:

- **«не удалось» у части заметок** — обычно битые файлы или экзотическая кодировка; остальной vault загружается, каждая проблемная заметка перечислена с причиной.
- **Vault не находится автоматически** — реестр Obsidian лежит в платформенной папке настроек (`%APPDATA%\obsidian\obsidian.json` на Windows); если Obsidian установлен нестандартно, просто укажите путь к vault руками, как в примере выше.

## 10. Гостевые коллекции

Зачем: пустить в свою базу другого человека (коллегу, клиента, семью) — но не во всю, а в отдельную «полку», с квотами и правами. Все команды ниже — только для владельца базы (`memobase.owner_user_id`; владельцем становится первый, кто прошёл `/memobase setup`).

```
/memobase create-for 123456789 docs-ivan
/memobase share docs-common 123456789 read
/memobase share-revoke docs-common 123456789
```

- `create-for` — создать гостю личную коллекцию (он сможет и загружать в неё, и спрашивать).
- `share ... read|write` — выдать доступ к существующей коллекции: `read` — только спрашивать, `write` — ещё и загружать.
- `share-revoke` — мгновенно отозвать доступ; сама коллекция и её содержимое остаются.

Квоты защищают ваш кошелёк: по умолчанию гость ограничен объёмом (200 МБ, 4000 фрагментов), дневной загрузкой (50 МБ), дневным бюджетом ($0.50) и числом вызовов (200/день, не чаще 6 в минуту). Поменять для конкретного гостя — из терминала:

```
hermes memobase set-guest-quota 123456789 --daily-budget-usd 2
```

(или просьбой в чате обычным текстом: «подними гостю 123456789 дневной бюджет до $2» — модель вызовет `memobase_set_guest_quota`).

Карантин: если загруженный гостем фрагмент похож на внедрение инструкций («игнорируй предыдущие команды...»), он не индексируется молча, а попадает в очередь на вашу проверку:

```
/memobase quarantine
hermes memobase quarantine-review 42 approve   # или reject
```

Что вы увидите: `/memobase quarantine` — список задержанных фрагментов с номерами и причиной; `approve` индексирует фрагмент, `reject` отклоняет.

Типовые ошибки:

- **«только владелец...»** — команду вызвал не владелец базы. Это проверяется в коде, а не «по договорённости» с моделью.
- **Гость упёрся в квоту** — он увидит честное сообщение о превышении; поднимите лимит через `set-guest-quota` или дождитесь следующего дня.

## 11. Мастер настройки: `/memobase setup` (Telegram) и `hermes memobase setup` (терминал)

Самый простой способ поднять базу с нуля — пройти мастер настройки, а не редактировать `.env` руками. У мастера **два входа, но один общий код** (`setup_core.py`): вопросы, проверка формата ключа, живая проверка у провайдера и маскировка ключа в выводе — везде одни и те же, поэтому оба пути ведут себя идентично и не могут разойтись друг с другом.

- **В Telegram** — отправьте `/memobase setup` и дальше просто отвечайте обычными сообщениями, по одному вопросу за шаг. Ответы мастера перехватываются **до** модели: токены не тратятся, ключи не попадают в контекст модели. Мастер доступен только владельцу; если владелец ещё не назначен — им становится первый, кто напишет `/memobase setup`. Состояние диалога хранится в `<HERMES_HOME>/memobase/wizard_state.json` — мастер переживает перезапуск hermes и продолжит с того же шага.
- **В терминале** — выполните `hermes memobase setup`. Тот же самый порядок вопросов и та же проверка ключей, но подряд в одной консольной сессии, без сохранения состояния между шагами (в терминале это и не нужно — сеанс не прерывается перезапуском hermes). Проверка «только владелец» здесь тоже не нужна: терминал по определению — привилегированный локальный оператор, у которого и так есть доступ к диску и `.env`. Команда `/memobase setup`, набранная не в Telegram, а в обычном CLI-чате, просто подскажет вам выполнить `hermes memobase setup`.

Шаги (одинаковые для обоих путей):

0. **Проверка окружения.** Ещё до первого вопроса мастер проверяет, установлен ли `ffmpeg` — он нужен, чтобы резать на куски длинные аудио и видео при расшифровке (STT, раздел 8). Если `ffmpeg` не нашёлся, мастер честно предупредит и подскажет команду установки (см. ниже) — это не блокирует настройку, остальные источники (файлы, YouTube с субтитрами и т.д.) прекрасно работают и без ffmpeg.
1. **Где считать эмбеддинги** — мастер спрашивает: `1` — на этой машине, полное качество (BGE-M3); `2` — на этой машине, полегче; `3` — через облако. Рядом подсказка по вашей RAM. Практический совет: сегодня рабочий вариант — `3` (облако); варианты 1–2 записывают выбор «локально» под будущий локальный движок, которого в этой версии ещё нет (см. Roadmap в README).
2. **Провайдер облака** — `1` Cloudflare Workers AI, `2` Cohere, `3` OpenAI-совместимый.
3. **Ключ** — пришлите API-ключ (в Telegram — отдельным сообщением; в терминале — строкой в ответ на приглашение). Он проверяется на формат (в том числе — не перепутан ли с ключом другого сервиса) и записывается сразу в `<HERMES_HOME>/.env` (права 0600), не сохраняется в истории модели; в Telegram, где возможно, ещё и просят удалить сообщение с ключом из чата. Мастер тут же проверяет ключ живым мини-вызовом и говорит: «ключ принят, эмбеддинг-провайдер отвечает ✓» — или честно, что проверить не удалось.
4. **Obsidian** — мастер сам сообщает, нашёл ли на машине vault'ы (по реестру Obsidian), и подсказывает, как их подключить позже.
5. **Первая загрузка** — пришлите файл или путь (Telegram) / укажите путь (терминал).
6. **Контрольный вопрос** — задайте любой вопрос по загруженному и получите первый ответ с цитатой.
7. **Готово** — шпаргалка команд.

### `ffmpeg` — зачем и как поставить

`ffmpeg` нужен только для расшифровки аудио и видео: без него длинные записи (больше 25 МБ) не получится разрезать на куски для Groq Whisper, а из видеофайла не получится вытащить звуковую дорожку. Мастер сам это заметит, но поставить `ffmpeg` за вас не может — как и в принтер программа для печати документа не встанет сама, если его нет рядом с компьютером. Поставить:

- **Windows:** `winget install Gyan.FFmpeg` (или `choco install ffmpeg`), затем перезапустите терминал/hermes.
- **macOS:** `brew install ffmpeg`.
- **Linux (Debian/Ubuntu):** `sudo apt install ffmpeg` (или пакетный менеджер вашего дистрибутива).

После установки достаточно перезапустить hermes — плагин сам находит `ffmpeg` в PATH.

### Самый простой путь для новичка

Если не хочется разбираться, какие именно ключи нужны MemoBase (их семь) — включите плагин `hermes-setup` один раз (`plugins.enabled: [hermes-setup]`) и отправьте `/setup` в Telegram: он объяснит каждый плагин простыми словами, включит его и настроит по очереди MemoBase и остальные, спрашивая ровно те ключи, которых не хватает.

Типовые ошибки:

- **«Мастер настройки доступен только владельцу»** — относится к Telegram-пути: `/memobase setup` написал не тот, кто владелец базы (`memobase.owner_user_id`). В терминале этой проверки нет.
- **«не удалось проверить ключ живым вызовом»** — ключ сохранён, но мини-проверка не прошла: проверьте ключ у провайдера и `/memobase status`.
- **«ffmpeg не найден»** — не блокирует настройку: расшифровка длинных аудио и видео будет недоступна, пока не поставите `ffmpeg` (см. выше), остальные источники работают как обычно.

## 12. Бэкапы

Бэкап — это консистентный снимок всей базы (`VACUUM INTO` — безопасный способ для SQLite в WAL-режиме, в отличие от простого копирования файла). Запускается из терминала:

```
hermes memobase backup-run
```

Что вы увидите: отчёт по шагам — снимок сделан, старые копии удалены по ротации, состояние диска, внешняя копия (если настроена).

- **Где лежат:** `<HERMES_HOME>/memobase/backups/memobase-ГГГГммдд-ЧЧММСС.db` (время UTC, имена сортируются по дате). Другую папку можно указать флагом `--dir`.
- **Сколько хранится:** последние `memobase.backup.keep` снимков (по умолчанию 7), старые удаляются автоматически.
- **По расписанию:** сам плагин расписание не ведёт — повесьте команду на штатный планировщик (`hermes cron` → `hermes memobase backup-run`, либо системный cron/Планировщик задач).
- **Копия вне машины:** задайте `memobase.backup.off_vps_command` — shell-шаблон с плейсхолдером `{snapshot_path}`, например `rclone copy {snapshot_path} remote:memobase-backups`.
- **Как восстановить:** снимок — это готовый файл базы. Остановите hermes, замените `<HERMES_HOME>/memobase/memobase.db` нужным снимком (старый файл предварительно переименуйте на всякий случай), запустите hermes и проверьте `/memobase status`. Команды «восстановить одним действием» пока нет — см. Roadmap.

Типовые ошибки:

- **Предупреждение о диске** — диск заполнен сильнее `memobase.backup.disk_alert_pct` (80% по умолчанию): почистите место или уменьшите `keep`.
- **Внешняя копия не ушла** — проверьте `off_vps_command` руками в терминале, подставив путь к снимку вместо `{snapshot_path}`; у команды есть таймаут 10 минут.

## 13. Карта базы (mind-map)

Карта — быстрый взгляд «что вообще лежит в коллекции и как оно связано»:

```
/memobase map
/memobase map docs
```

Что вы увидите: mermaid-диаграмму (текстовый блок ```` ```mermaid ````). В клиентах с поддержкой mermaid (Obsidian, GitHub, многие IDE) она рендерится картинкой; в Telegram придёт как код — вставьте его в любой mermaid-просмотрщик (например, mermaid.live).

Как читать:

- **узлы** — документы коллекции с их темами (топ-5 ключевых слов, извлекаются по частотности, без затрат на модель);
- **сплошные стрелки** — явные Obsidian-связи `[[...]]` между документами;
- **пунктирные подписанные стрелки** — тематическая близость: у документов совпали 2+ ключевых слова.

Ограничения: в карту попадает не больше 60 документов (об усечении будет пометка); настроек глубины нет. На пустой коллекции вместо графа придёт честное пояснение.

## 14. Диагностика

**`/memobase status` показывает «векторный индекс недоступен (sqlite-vec не установлен)»**
Пакет `sqlite-vec` не встал при установке (например, нет подходящего wheel для вашей платформы/версии Python). Поиск продолжает работать, но только по полнотекстовому (FTS5) индексу, без векторной части. Проверьте вывод `install.ps1`/`install.sh` на ошибки и попробуйте переустановить пакет вручную в venv hermes-agent.

**Ответы всегда «деградированный режим»**
Проверьте `COHERE_API_KEY` в `~/.hermes/.env` — при ошибке авторизации Cohere плагин не падает, а тихо переходит на RRF-only ранжирование с отдельным, более консервативным порогом. То же самое для Cloudflare: неверные `CLOUDFLARE_ACCOUNT_ID`/`CLOUDFLARE_API_TOKEN` не уронят загрузку, но эмбеддинг не будет работать — проверяйте эти три ключа в первую очередь.

**«Нужно подтверждение» при загрузке**
Новых фрагментов больше порога `memobase.confirm_over_chunks` (по умолчанию 500). Это защита от случайной дорогой загрузки. Повторите загрузку с подтверждением (`confirm=true` из модели, `--confirm` из терминала) — оценка стоимости уже показана в сообщении.

**«Заблокированы сканером секретов»**
Часть фрагментов содержала похожее на секрет содержимое (ключ API, токен, приватный ключ, строка с высокой энтропией) и не была проиндексирована. Проверьте документ вручную, уберите секрет и загрузите заново — либо, если это ложное срабатывание на случайную строку, знайте, что это осознанный компромисс безопасности (сканер сделан блокирующим по спецификации).

**«База знаний сейчас переиндексируется»**
Коллекция в процессе смены модели эмбеддингов (`hermes memobase reindex <коллекция>`, запускается только из терминала оператором) — ответы временно недоступны, дождитесь завершения.

**Достигнут месячный потолок расходов**
`memobase_ingest` вернёт ошибку загрузки с упоминанием потолка (`memobase.monthly_ceiling_usd`). Поднимите потолок в конфиге или дождитесь начала следующего 30-дневного окна; `/memobase status` всегда показывает текущий расход против потолка по каждому провайдеру.

**Нет ответа от `memobase_ask`, ошибка «нет доступа к модели»**
`ctx.llm` не инициализирован на момент вызова — это ограничение окружения хоста hermes на данный момент, а не конфигурация плагина; `memobase_query` (сырые фрагменты без генерации ответа) при этом продолжает работать.

**YouTube/аудио/Obsidian «пропущено» при загрузке**
Это штатная деградация, а не сбой: у каждого пропуска есть причина в ответе. Чаще всего — отсутствует ключ (`SCRAPECREATORS_API_KEY`, `APIFY_TOKEN`, `GROQ_API_KEY`) или для аудио не найден `ffmpeg`. Разделы 7–9 выше разбирают каждый источник.

**`/memobase ingest ... youtube` отвечает «Неизвестный тип источника»**
Так и задумано: слэш-команда принимает только файловые типы и URL. YouTube, аудио и Obsidian загружаются просьбой в чате обычным текстом — модель вызовет `memobase_ingest` с нужным типом сама (разделы 7–9).

**Гость получает отказы на каждый запрос**
Проверьте квоты: `/memobase status` покажет расход, а лимиты гостя задаются `hermes memobase set-guest-quota` (раздел 10). Отказ с формулировкой про владельца означает, что гость пытается вызвать owner-only операцию — это нормально.

**Загрузка от гостя «исчезла»**
Загляните в `/memobase quarantine` — фрагменты, похожие на внедрение инструкций, задерживаются до вашей ручной проверки (`hermes memobase quarantine-review <id> approve|reject`), об этом же сообщается в ответе на загрузку.
